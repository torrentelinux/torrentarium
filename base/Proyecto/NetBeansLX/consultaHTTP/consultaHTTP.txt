'consultaHTTP.txt' contiene una documentación técnica básica sobre
el programa 'consultaHTTP.java'.

La aplicación 'consultaHTTP.jar' hace una consulta sobre el servicio de información
ofrecido por el servidor http/https y muestra en pantalla los datos recibidos desde
el sitio consultado.

Este proyecto 'consultaHTTP' es para NetBeans (Linux/Windows).
Se utilizó la biblioteca Java 25 zulu (openjdk+jdkfx) y Apache NetBeans IDE 25.

Importante: No declarar "package ejemplohttpcliente;" en el código fuente sino
usar la opción --enable-preview.
Poner esta opción en:
     'Project Properties\Run\VM Options'
     'Project Properties\Build\Compiling\Additional Compiler Options'

Ejecutar el programa desde la línea de comandos:
    java -jar dist/consultaHTTP.jar

y seguir los consejos que aparecen en la pantalla.
Para conocer el código devuelto por la aplicación en CMD, ejecute:
    1) java -jar dist\consultaHTTP.jar www.google.com
    2) echo %ERRORLEVEL%

O también, así:
    java -jar dist\consultaHTTP.jar www.google.com & echo %ERRORLEVEL%

Para conocer el código devuelto por la aplicación en TCC, ejecute:
    echo %(java -jar dist\consultaHTTP.jar www.google.com)

Mostrará un valor 0 (Ok) o 3 (No hay conexión a Internet).
Es de destacar que con el valor devuelto por 'consultaHTTP.jar' puede deducir Usted
si tiene conexión o no a Internet en su computadora.

Se probó en TCC ver. 35.00.21, Windows 64 bits.
TCC (Take Command Console) es un intérprete de comandos que corre en Windows.
Su sitio en Internet es 'jpsoft.com' y allí buscar la página para descargar una
versión de evaluación.

En caso de error en el uso de Java JVM, por ejemplo:
Error: LinkageError occurred while loading main class consultaHTTP
    java.lang.UnsupportedClassVersionError: consultaHTTP has been compiled by a more
    recent version of the Java Runtime (class file version 69.0), this version of the
    Java Runtime only recognizes class file versions up to 61.0

La versión 69 corresponde a Java JDK 25 y la versión 61 corresponde a Java JDK 17.
Esto se debe a que se necesita de Java SDK 25 para correr/ejecutar la aplicación
'consultaHTTP.jar'
Busque y descargue en Internet un JDK versión 25 o superior luego instale dicho
paquete en su S.O. (Linux/Windows).

Otros casos de usos pueden ser examinar su celular, enrutador local de la casa, la tablet, etc.
Si se va a hacer un control de conectividad en cada dispositivo de su casa, todos ellos deberían
estar conectados al mismo enrutador local y proceder luego a su control interno.

Si desea correr un servidor http/https en su computadora le sugiero 'jwebserver'.
Esta aplicación viene incluído en el paquete Oracle Java/SDK y en Java/OpenJDK.
Para conocer sus diferentes opciones, ejecute 'jwebserver -h' en la consola
de textos de su S.O. y en Linux ejecute 'man jwebserver' para ampliar su consulta.
Este servidor abre el puerto TCP/8000 y ofrece servicios básicos de http.
Por ejemplo, ejecute en Linux:
        jwebserver -d /base/sitio -o verbose -p 80 &

Se habilita el servicio http en el puerto TCP/80 y el directorio '/base/sitio' es expuesto para su acceso.
En dicho directorio ponga el archivo 'index.html' con algún texto de su gusto.
Después, ejecute 'consultaHTTP.jar localhost' para conocer los datos recibidos del servidor http.
Aclaro: primero crear el directorio '/base/sitio/' y luego crear el archivo 'index.html' en dicho directorio.
